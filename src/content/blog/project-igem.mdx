## 竞赛简介

iGEM（International Genetically Engineered Machine）是全球性的合成生物学竞赛。在 2025 年 USTC 队项目中，我担任网页组组长，主要负责 Wiki 网站的架构设计、页面开发、内容接入与协作推进

![iGEM Team](/images/blog/project-igem/igem_team.png)

<Callout type="info" title="项目交付背景">
  这次项目的核心交付物并不是一个泛化的“比赛官网”，而是一套严格面向 iGEM Wiki
  展示与评审需求的静态网站工程。我们需要在有限的开发周期内，既保证视觉与交互的高质量，又要确保内容组能便捷地填充大量实验与建模文档
</Callout>

## 核心工程架构

为了满足静态部署与高性能访问的需求，我们构建了一套基于 Next.js 的现代化工程方案

### 1. 技术栈选型

项目基于 **Next.js 15** 和 **React 18** 构建，既保留组件化开发体验，也方便后续做静态导出。在样式与交互层面，我们采用了混合技术栈以兼顾开发效率与表现力：

- **样式层**：SCSS Modules 用于组件级样式隔离，结合 Tailwind CSS 处理原子化样式
- **交互层**：引入 GSAP 处理复杂时间轴动画，Lenis 优化滚动体验，Lottie 负责矢量动画渲染
- **工程化**：集成 ESLint、Prettier、Husky 保证多人协作时的代码规范

### 2. 静态自动化部署

针对 iGEM 比赛服务器的特性，我们在生产环境下配置了纯静态导出模式，并解决了路径前缀问题

<Details title="查看 Next.js 静态导出配置逻辑" client:load>

```javascript
// next.config.js 核心配置片段
const isProduction = process.env.NODE_ENV === "production";

const nextConfig = {
  output: isProduction ? "export" : undefined, // 生产环境启用静态导出
  distDir: isProduction ? "out" : ".next",
  // 适配比赛服务器的子路径部署
  basePath: "/ustc",
  assetPrefix: isProduction ? "/ustc" : "",
  // 静态导出模式下需禁用服务端图片优化
  images: {
    unoptimized: isProduction,
  },
};
```

</Details>

## 关键功能实现

### Markdown 驱动的内容管线

为了让内容组能专注于文档编写，我们设计了一套将 Markdown 直接转化为 React 页面的自动化管线，实现了内容与代码的解耦

<Step number={1} title="自定义 Loader 接入">
  在 `next.config.js` 中配置本地 `md-loader`，拦截 `.md` 文件加载请求，通过
  Babel 将 Markdown 内容转化为 React 可渲染的组件结构
</Step>

<Step number={2} title="运行时渲染与解析">
  开发 `local-md-loader.js`，在前端运行时动态拉取 `static/md`
  下的文档资源，并交给 `MDRenderer` 进行解析，支持公式、代码高亮等富文本特性
</Step>

<Step number={3} title="自动化目录生成">
  页面组件（如 `description.jsx`）在渲染后会自动扫描文档中的 section
  结构，提取标题层级，自动生成侧边导航目录，彻底告别了手动维护锚点的繁琐工作
</Step>

### 分层加载与性能优化

为了提升 Wiki 在评审过程中的首屏体验，我们设计了一套完整的分层加载机制

- **全局状态管理**：在 `_app.jsx` 中统一挂载 Theme、Page、Interaction 等 Context，确保全站状态共享
- **关键资源预加载**：应用启动时优先加载核心样式与脚本，通过 `GlobalLoader` 遮罩过渡，避免样式闪烁
- **分级加载策略**：非关键样式与重型组件通过动态导入（Dynamic Imports）延后加载，确保主内容优先可见

### 页面组织与导航系统

Wiki 的页面结构严格遵循比赛评审逻辑，并未采用单页堆砌，而是建立了清晰的路由体系：

- **路由模块化**：`src/pages` 下按 `description`、`design`、`results` 等模块拆分独立路由
- **配置化导航**：通过 `navigation.js` 统一管理 Project、Wet Lab、Dry Lab 等五大版块的菜单结构
- **统一布局模式**：利用 `getLayout` 模式为不同页面注入统一的页头、页脚与背景逻辑，同时保留了局部页面的自定义能力

## 团队协作与管理

作为网页组负责人，我的工作重心不仅是代码实现，更在于构建高效的协作流

- **资源自动化同步**：编写 `lychee-sync` 脚本，在构建前自动同步图片素材到静态目录，减少了手动搬运的机械劳动
- **组件化开发**：将通用的图表、弹窗、卡片封装为独立组件，让组员只需关注业务逻辑，大幅降低了代码耦合度
- **需求分层推进**：在内容未定稿时先行搭建骨架屏与路由结构，确保开发进度不被内容产出阻塞
