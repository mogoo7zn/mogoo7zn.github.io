# 摘要（Abstract）

手物交互（Hand-Object Interaction, HOI）是空间智能中的一个核心问题。对于人类而言，观察当前场景、想象目标状态，并进一步推断出完成这一状态变化所需的手部动作，是一种自然的能力；但对于计算模型来说，要同时兼顾视觉真实性、空间一致性与物理合理性，仍然十分困难。

现有方法通常分别从 2D 或 3D 的角度处理 HOI 生成问题：基于 2D 的方法在外观生成上具有优势，能够较好地恢复纹理、光照与场景上下文，但往往缺乏显式几何约束，容易产生手部结构错误、交互区域偏移以及遮挡关系不合理等问题；基于 3D 的方法虽然更强调接触关系与几何结构，但通常难以利用丰富的视觉先验，在复杂场景中容易缺少全局上下文理解，生成结果也较难达到较高的视觉保真度。

针对这一问题，我们提出 **THREAD**，用于解决 **第一视角手物交互的联合 2D-3D 生成**。该任务的目标是：在给定当前观察以及物体当前/目标状态的条件下，同时生成目标时刻对应的 **2D 第一视角视觉结果** 与 **3D 手物交互结构**。与传统串联式流程不同，THREAD 将 2D 视觉生成与 3D 几何生成置于统一框架中，通过双向耦合实现相互约束：2D 分支提供视觉与语义先验，3D 分支提供空间与接触约束，二者共同优化，最终在视觉真实性与物理合理性之间取得更好的平衡。

## 研究背景（Background）

在具身智能场景中，智能体不仅需要“识别当前看到的内容”，更需要进一步理解“为了达到目标状态，应该如何与环境发生交互”。这类问题并不只是动作预测，也不只是图像生成，而是同时涉及未来状态想象、空间关系推理和交互可供性判断。

近年来，大规模图像与视频生成模型在视觉合成上取得了显著进展，使得模型具备了更强的外观建模能力；与此同时，HOI 领域中的 3D 方法也在尝试从接触、抓取与几何结构的角度对手部动作进行建模。然而，这两类能力目前仍然大多是分离的：视觉模型擅长“生成看起来合理的结果”，几何模型擅长“保证结构上成立的交互”，但如何在一个统一框架中把这两种能力结合起来，仍然是一个开放问题。

THREAD 的工作正是在这一背景下展开。本文关注的是第一视角场景中的手物交互生成，希望让模型不只是在图像空间中“画出一个手”，也不只是在几何空间中“拟合一个姿态”，而是能够面向目标状态，同时生成视觉上自然、空间上准确、交互上合理的手物交互结果。

---

## 问题定义（Task Formulation）

本文形式化了一个新的任务：**Joint 2D-3D Egocentric HOI Generation**。

具体而言，给定当前时刻的第一视角观测、当前交互状态以及目标物体状态，模型需要预测目标时刻的两类结果：

1. **2D 视觉输出**：目标时刻从第一视角观察到的手物交互图像；
2. **3D 结构输出**：与该图像相对应的三维手部交互结构。

这一任务与传统图像翻译、视频预测或 3D 姿态估计都不完全相同。它要求模型同时解决两个问题：一是目标时刻“看起来会是什么样”，二是“在三维空间中手应该如何运动并与物体接触”。只有这两个问题同时回答得合理，生成结果才真正具有可用性。

---

## 核心困难（Challenges）

THREAD 主要针对以下三类困难展开设计。

### 1. 2D 与 3D 之间存在循环依赖

联合生成过程中，3D 结构的生成通常需要来自 2D 的视觉线索，例如交互区域、场景布局和目标外观；而 2D 图像的生成又需要来自 3D 的空间条件，例如深度、接触位置和几何关系。也就是说，2D 依赖 3D，3D 也依赖 2D，这使得系统一开始很难稳定启动。

### 2. 2D 与 3D 各自存在互补性的缺陷

纯 2D 方法通常缺乏显式空间约束，因此难以保证手和物体之间的几何一致性；纯 3D 方法则更容易退化为局部接触优化，缺少对场景上下文和目标视觉结果的整体理解。要得到既“看起来正确”又“结构上成立”的结果，必须让这两种信息真正协同起来。

### 3. 高维手部参数优化代价高且不稳定

如果直接在参数化手模型（如 MANO）的高维空间中优化，通常会遇到计算开销大、训练不稳定、容易陷入局部最优等问题。尤其是在联合训练时，这种问题会进一步加剧。因此，需要找到一种既保留结构约束、又便于优化的中间表示。

---

## 方法概述（Method Overview）

为了解决上述问题，本文提出了 **THREAD**。整个框架的核心思想是：在 2D 视觉分支与 3D 结构分支之间建立一个双向耦合的生成闭环，使视觉先验和几何约束能够互相传递，并在联合训练中相互增强。

从整体流程上看，THREAD 包含三个关键模块：

1. **Prior-Guided 2D State Synthesis and Hotspot Extraction**  
   先利用视觉先验进行冷启动，并从 2D 分支中提取交互热点；
2. **Heatmap-to-Contact Lifting**  
   将 2D 交互热图提升到 3D 接触表示，形成显式几何约束；
3. **Efficient Point-Cloud-Based Hand Motion Generation**  
   在点云空间中生成手部交互结构，再高效恢复为参数化手模型。

在此基础上，THREAD 进一步通过双向 2D-3D 耦合与联合优化，将两个分支整合到统一训练框架中。

## 方法细节（Method Details）

## 1. 先验引导的 2D 状态生成与热点提取

为了打破 2D 与 3D 之间的循环依赖，THREAD 首先引入一个 **冷启动模块**。这一模块利用预训练视觉模型 **DINOv2**，从初始观测图像和目标物体渲染图中提取多尺度视觉特征，并预测一个细粒度的 **2D 交互热图**。这个热图并不直接给出完整手部结构，而是先回答一个更基础的问题：在目标状态下，手更可能与物体的哪些区域发生交互。

这种设计的作用在于，它为后续的 3D 推理提供了显式的交互先验，使系统在联合生成的一开始就获得了比较稳定的空间提示，从而缓解冷启动问题。

除了语义先验分支，THREAD 还构建了一个 **生成先验分支**。该分支基于潜空间扩散模型，在当前图像、粗交互渲染图以及对应深度图的条件下生成目标图像。同时，模型在 UNet 解码器上附加 FPN 头，从不同分辨率层中聚合特征，进一步预测交互热图。这样，系统不仅能生成目标视觉外观，还能从生成过程本身中提取与接触相关的空间信息。

简而言之，这一部分同时利用了两类 2D 先验：
- 一类是来自预训练视觉编码器的**空间语义先验**；
- 一类是来自扩散生成模型的**外观生成先验**。

二者分别承担冷启动与高保真生成的职责，并在热图层面实现汇合。

---

## 2. 从 2D 热图到 3D 接触表示的提升

仅有图像平面中的热图仍然不足以驱动可靠的 3D 手部生成，因为 3D 分支需要的是可优化的显式空间约束。为此，THREAD 设计了 **Heatmap-to-Contact Lifting** 模块，将 2D 交互概率提升到 3D 接触表示中。

该模块基于 **Basis Point Set（BPS）** 对物体进行几何编码。BPS 的基本思想是使用一组固定基点来表征物体表面结构，并以这些基点与物体表面的距离来构造紧凑、统一的几何表示。对于关节式物体，不同部件可以分别建立局部 BPS 表示，从而更自然地处理复杂物体的结构变化。

在 THREAD 中，模型会先将这些基点投影到图像平面，再根据 2D 热图读取对应位置的交互概率。这样，每个 BPS 基点都获得了一个与接触可能性相关的权重。随后，模型通过一个基于 **FiLM** 的残差预测器，在当前接触距离的基础上预测目标状态下的接触距离变化。最终得到的结果不再只是“图像上的热点”，而是一个可直接用于 3D 推理的 **接触距离场表示**。

这一设计非常关键，因为它在 2D 和 3D 之间建立了一条清晰的桥梁：  
图像中的交互线索被转化为几何空间中的显式约束，后续 3D 分支不再需要在完全无引导的高维空间中搜索合理解。

---

## 3. 基于点云的 3D 手部交互生成

在 3D 分支中，THREAD 没有直接在 MANO 参数空间中生成手部结构，而是先在 **语义点云空间** 中进行建模。具体来说，模型将手部动作表示为一组稀疏手部表面关键点，以及这些点到物体表面的方向向量。这样可以把手与物体放在同一个物体中心坐标系下统一描述，也更容易表达交互关系。

在这一表示基础上，THREAD 使用一个 **Transformer-based diffusion denoiser** 对手部动作状态进行扩散建模。它接收来自热图提升模块的空间约束、物体的 BPS 特征以及全局物体状态作为条件，直接预测无噪动作状态。

点云生成完成后，系统还需要恢复为标准的参数化手模型表示。传统做法通常依赖迭代拟合，这在联合训练中会带来很大的计算负担。为此，THREAD 设计了一个 **语义锚点驱动的解析逆运动学（Analytical IK）** 过程。该方法先从语义点云中选择具有解剖意义的锚点，包括指骨环、关节/指尖对、腕部锚点和手掌表面锚点等，再通过单步解析估计的方式依次恢复尺度、全局朝向、局部关节旋转和根平移。

这一设计有两个明显优点：
- 它避免了传统迭代拟合的高成本；
- 它保留了解剖结构与运动学链条中的显式约束。

因此，3D 分支既具备较强的表达能力，也保持了较好的训练与推理效率。

## 双向 2D-3D 耦合与联合优化（Bidirectional Coupling）

THREAD 的关键不在于简单地把 2D 和 3D 两条流水线串起来，而在于建立一个真正双向的信息闭环。

一方面，3D 分支生成的粗 RGB 渲染图和深度图会作为 2D 扩散分支的条件输入。这些显式几何条件弥补了 2D 生成中空间约束不足的问题，使目标图像在遮挡关系、接触位置和整体布局上更稳定。

另一方面，在联合训练阶段，THREAD 通过 **Score Distillation Sampling（SDS）** 将 2D 扩散模型作为视觉先验反向作用于 3D 分支。具体来说，3D 分支生成的手部交互结果被渲染为图像后送入 2D 扩散模型，后者给出与视觉分布一致性的梯度信号，从而推动 3D 结构朝着更符合真实视觉规律的方向优化。

这种双向耦合意味着：
- 2D 分支不再只是最终“渲染结果”的输出端，而是主动参与 3D 结构优化；
- 3D 分支也不再只是中间约束，而是持续为 2D 生成提供空间条件。

在训练策略上，THREAD 采用渐进式方案：先分别训练冷启动模块、2D 生成模块和 3D 结构模块，再在此基础上进行统一联合优化。这样既能保证各个子模块先获得稳定能力，也有助于后续端到端协同收敛。

## 实验设置与结果（Experiments and Results）

本文主要在 **ARCTIC** 数据集上进行系统评测，并在 **WildO2** 数据集上测试方法在开放场景下的泛化能力。

在 ARCTIC 上，作者构建了基于接触状态变化的训练样本对，用于模拟从当前状态到目标状态的交互转移过程。模型在 2D 图像质量和 3D 交互质量两个维度上分别进行评估。2D 部分采用 PSNR、SSIM、LPIPS、FID、sFID 等指标；3D 部分采用 MPVPE、MPJPE、Chamfer Distance、Contact Ratio、Penetration Depth、Penetration Volume 等指标。

实验结果表明，THREAD 在两方面都取得了较好的表现：

### 2D 结果
在 ARCTIC 上，THREAD 的 2D 生成结果优于 HOIDiff、BimArt、CosHand 等对比方法，在 PSNR、SSIM、LPIPS、FID 和 sFID 等指标上都取得了最优结果。特别是在视觉保真度与感知质量方面，联合几何条件后的 2D 分支表现出更强的稳定性和更好的场景一致性。

### 3D 结果
在 3D 手部交互评测中，THREAD 在 MPVPE、MPJPE、Chamfer Distance 以及 Contact Ratio 上同样优于现有方法。结果表明，联合 2D 先验后的 3D 分支不仅几何误差更小，而且更容易形成有效接触关系。

### 消融实验
论文还进一步验证了多个关键设计的作用，包括：
- 去除冷启动模块后，2D 结果明显退化；
- 相比法线图等几何条件，深度图对 2D 生成的帮助更明显；
- 以点云作为中间表示并采用单步解析 IK，能够在几乎不损失精度的情况下大幅提升效率；
- 在 MANO 参数空间中直接优化不如先生成点云再恢复结构稳定。

### 开放场景泛化
在 WildO2 上，尽管目标物体与真实交互帧之间的状态差异更细微、背景更复杂，THREAD 仍然能够生成较为可信的 2D/3D 联合结果，说明方法具有一定的开放场景泛化能力。

## 工作意义（Why This Work Matters）

这项工作的价值并不只在于提出一个新的 HOI 生成框架，更在于它尝试把几个原本彼此分离的问题放到同一个系统中统一处理：  
视觉想象、接触推理、空间建模和结构恢复，不再分别由不同模块松散串联，而是通过双向耦合共同作用于最终结果。

从具身智能的角度看，这一问题比单纯的图像生成更接近真实交互需求。一个智能体若想完成操作任务，既需要知道目标状态下场景“会长什么样”，也需要知道“手应该怎样动、在哪里接触、这个接触在三维空间中是否可行”。THREAD 试图同时处理这两件事，因此它不仅适用于数据生成和 HOI 建模，也为后续更长时序的操作预测、视频生成、4D 交互建模以及可供性推理提供了一个较自然的基础框架。

---

## 局限性与未来工作（Limitations and Future Work）

尽管 THREAD 已经在联合 2D-3D HOI 生成上取得了较好的结果，但仍存在一些限制。

首先，当前框架主要面向短时、离散状态转移建模，对于长时序、多步操作过程的建模仍不充分。其次，方法对复杂可变形物体和更高自由度交互的扩展能力还有待进一步研究。再次，由于 2D 与 3D 分支的耦合方式仍以中间条件和先验约束为主，最终 2D 输出与 3D 结构之间仍可能出现一定程度的不严格对齐。

这些问题也为后续工作提供了明确方向：一方面，可以引入显式时序模块，把当前框架自然扩展到视频或 4D 交互生成；另一方面，也可以进一步强化 2D-3D 一致性约束，使视觉结果与几何结构之间建立更严格的对应关系。
