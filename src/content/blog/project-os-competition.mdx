# 全国大学生操作系统竞赛

## 赛题背景

我们的目标是在 DAYU200 这样的资源受限设备上，挑战大语言模型的高效部署。这不仅是对技术的考验，也是对如何在有限资源下最大化性能的探索

## 技术方案

### 模型选型与优化

为了在端侧跑起来，我们做了很多尝试：

- **基座选择**：选用了 Llama 系列模型，平衡了性能与体积
- **框架迁移**：利用 MNN 推理框架进行模型的轻量化转换与深度优化
- **极限压缩**：通过 INT4/INT8 权重量化技术，大幅降低了显存占用

### GPU 加速

不仅仅依靠 CPU，我们还充分挖掘了 DAYU200 的 GPU 潜力：

- **OpenCL 适配**：重写了算子后端，让计算更贴合硬件特性
- **内存优化**：精细化管理显存申请与释放，减少不必要的开销
- **算子融合**：合井多个计算步骤，减少内存访存次数，提升推理速度

### 多模态探索

在决赛阶段，我们向更高难度发起挑战，探索多模态大模型与操作系统的协同：

- **视觉轻量化**：重新设计视觉编码器，使其适应端侧算力
- **流式处理**：实现了多模态输入的流式处理，降低延迟感
- **最终成果**：成功实现了约 15 token/s 的端侧推理能力，带来了流畅的交互体验

## 队长经历

作为队长，这段旅程不仅仅是技术攻关，更是团队协作的磨练。我深刻体会到，如何在压力下凝聚团队，合理分配任务，是比写代码更具挑战性的课题

## 核心收获

- 深入理解了 LLM 在端侧部署的各种优化黑科技
- 积累了在嵌入式设备上进行资源管理的宝贵经验
- 锻炼了带领团队攻克难题的项目管理能力
